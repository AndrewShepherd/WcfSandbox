Security Walkthrough

The goal is to create secure two-way communication using NetTcpBinding.

This will involve having a private key on the server, and a public key on the client

The steps, as I remember, as as follows:

  • Ensure we have a root certification authority set up.
  • Ensure that a revocation list is installed for this authority
  • Create a certificate that is signed by this root certificate authority, and load it into the certificate store
  • Tell the server to use it
  • Tell the client to expect it

I'm not sure how this will work using two-way NetTcpBinding

To demonstrate this, we will use a Chatroom application
